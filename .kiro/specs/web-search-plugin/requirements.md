# Web Search Plugin Requirements

## 1. 概要
現在スタブ実装となっている `web_search` プラグインを、外部検索APIと連携させて実際に機能するようにアップデートする。

## 2. 目標
- ユーザーが複数の検索APIプロバイダ（例：Tavily, Google Custom Search）から利用するものを選択できる。
- ユーザーが設定画面から自身のAPIキーやその他の認証情報（CX等）を入力・保存できる。
- チャット時に LLM が `web_search` を呼び出すと、選択されたプロバイダを通じて実際のWeb検索結果を取得し、LLMに返す。
- 検索で得たURLからページ本文を取得するための追加ツール `fetch_page` を実装し、LLM が自律的に深掘り検索できるようにする。
- `fetch_page` にはセキュリティ・プライバシー保護のため **ドメイン・ホワイトリスト機能** を実装する。ユーザーが許可したドメインにのみアクセス可能とし、それ以外はブロックする。
- APIキーが未設定の場合は、エラーを出力する代わりに「ユーザーに設定画面でAPIキーを入力するよう促す」テキストを LLM に返却し、自然な対話で案内できるようにする。

## 3. サポートする検索API
1. **Tavily Search API (推奨)**
   - LLM エージェント向けに最適化されており、クリーンなページ内容を直接抽出して返せるため相性が良い。
2. **Brave Search API**
   - 無料枠があり、登録しやすい標準的な検索API。Google Custom Searchの代替として採用する。
