# Workspace and UI Enhancements (file_ops, Vision, Persistence, Resizable Pane)

## 概要
チャットAIの柔軟性と利便性を向上させるため、以下の4つの主要な改善を行います。
1. **ディレクトリごとのRead/Write権限管理**: AI(file_opsプラグイン)がアクセスできるディレクトリパスを複数管理し、それぞれについて「読み取り(Read)」「書き込み(Write)」の許可を個別に設定できるようにする。
2. **UI右ペインの高度なコンポーネント化とテーマ対応**: 
   - 右ペインの幅をリサイズ可能にする。
   - ツール管理ペイン内で、プラグインごとの設定内容（ツールリスト、独自設定など）を「ツールごとの折り畳み（アコーディオン）UI」にまとめる。`file_ops` のディレクトリ管理もその中の一つとして配置する。
   - スクロールバーのダークモード対応および、ライト/ダークモード切り替えボタンを追加する。
3. **画像認識 (Vision) の対応**: file_opsプラグイン経由で画像ファイルを読み込み、LLMのVision機能を利用して画像内容を認識させる。
4. **プラグイン設定・許可状態の永続化**: チャット内で許可したディレクトリ権限やツール実行設定をデータベースに保存し、アプリ再起動後も状態を維持する。

## 目標 (ユースケース)
- ユーザーは右ペインのツール管理から `file_ops` を展開し、「`D:/project` はRead/Write許可」などの細かいアクセス制御リスト(ACL)を設定できる。他のプラグインも同様に折り畳んで管理できる。
- ユーザーはチャット内でAIに「このフォルダを管理対象に追加して」と指示し、AI側から権限リクエストを発行することもできる。
- ユーザーは許可したフォルダ内の画像ファイルをAIに処理させ、AIはその内容を理解して回答できる。
- アプリを再起動して同じチャットを開いた際、前回のディレクトリ権限設定が復元され、再度設定する必要がない。
- 画面サイズや好みに合わせて、右ペインの幅を自由にドラッグして調整できる。
- ユーザーの好みに応じてテーマ（ダーク/ライト）を切り替えられ、スクロールバーもそれに合わせて綺麗に表示される。

## 非機能要件
- **セキュリティ**: file_opsプラグインは、設定された複数の許可ディレクトリを横断して検証を行い、許可されていないパスやトラバーサル攻撃へのアクセスをブロックする。
- **パフォーマンス**: 画像のBase64エンコードやデータベースからの設定読み込みがチャットの応答性を著しく損なわないようにすること。
- **UI/UX**: リサイズやアコーディオンの開閉はスムーズであること。テーマの切り替えはUI全体に一貫して適用されること。
