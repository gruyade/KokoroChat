# LLM Tool Use Requirements

## 1. 概要
LLM が提供する Function Calling (Tool Use) 機能を活用し、チャット中に LLM がシステムのリソース（計算機、ファイル操作、Web検索など）を利用できるようにする。

## 2. 目標
- ユーザーが手動でツールやプラグインをグローバルおよびチャット（セッション）単位で有効化/無効化できる。
- ツール許可状態の優先順位は「グローバル不許可 ＞ チャット内不許可 ＞ グローバル許可 ＞ チャット内許可」。グローバルで不許可のツールはチャット内でも選択不可とする。
- ユーザー自身がカスタムツール（プラグイン）を追加・削除できること。ただし、デフォルトの組み込みツールは削除不可（無効化は可能）とする。
- ツールが単なるテキストではなく、チャット画面上でリッチなUI（地図、グラフ、カスタムウィジェット等）を描画できるような「カスタムUIコンポーネントレンダリング」の仕組みをサポートする。
- チャット内のツール管理UIとして、チャット画面右側に格納可能なペイン（サイドバー）を設置する。
- ツールの作成に関する仕様・スキーマを決定し、ドキュメント化する。
- LLM が必要と判断したタイミングで、自動的にツールを呼び出す。
- ツールの実行結果をチャットのコンテキストに含めて再度 LLM を呼び出し、最終的な回答をユーザーに提示する。
- 実行中のツール情報をユーザーに視覚的にフィードバックする。

## 3. 非機能要件
- ツール呼び出しと通常のテキストストリーミングがスムーズに連携すること。
- OpenAI, Anthropic, Gemini の各 API における Tool Call 形式の違いを吸収する。
- セキュリティのため、破壊的操作を伴うツールはユーザーへの事前確認または厳密なスコープ制限を設けること。
